## Contributing to Documentation

When updating documentation:

1. **Keep it current**: Update docs as part of code changes, not after
2. **Be specific**: Include examples, screenshots, and code snippets
3. **Think like a user**: Write for someone who doesn't know the system
4. **Test procedures**: Verify all deployment/operation procedures actually work
5. **Version appropriately**: Note which version documentation applies to

### Documentation Standards

- Use clear, concise language
- Include code examples where relevant
- Add diagrams for complex concepts
- Keep navigation intuitive
- Maintain consistent formatting
- Update SESSION_NOTES.md at end of each session
